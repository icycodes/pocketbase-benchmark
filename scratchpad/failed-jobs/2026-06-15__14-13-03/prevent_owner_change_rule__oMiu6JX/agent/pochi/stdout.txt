## How the Update Rule Works

```
owner = @request.auth.id && @request.body.owner:isset = false
```

- **Condition 1** (`owner = @request.auth.id`) — the record's current `owner` must equal the authenticated user's ID. Non-owners get a 404 because the rule filters out the record entirely.
- **Condition 2** (`@request.body.owner:isset = false`) — the `:isset` modifier returns `true` when the `owner` key is present in the request body. By asserting it must be `false`, any request that includes `owner` in the payload causes the entire rule to evaluate to false → 404 (blocked).
