# Custom JSON Route Handler (PocketBase JSVM) ## Background PocketBase (v0.31.0) ships with an embedded JavaScript engine (Goja) that lets you extend the server without writing Go. By dropping `*.pb.js` files into the `pb_hooks/` directory you can register custom HTTP routes that respond alongside the built-in REST API. In this task you will use this capability to add a small public greeting endpoint to a running PocketBase instance. The PocketBase server is already installed and configured to run from the project directory.
```js
routerAdd("GET", "/api/myapp/hello/{name}", function (e) {
    var name = e.request.pathValue("name");
    return e.json(200, { message: "Hello, " + name + "!" });
});
```

This registers a `GET /api/myapp/hello/{name}` route that:
- Reads the `{name}` path parameter via `e.request.pathValue("name")`
- Returns HTTP 200 with `{"message": "Hello, <name>!"}` using `e.json()`
- Has no auth middleware, so it's accessible to unauthenticated guests
- Uses only synchronous ES5.1-compatible code (no async/await/Promises)

The built-in PocketBase routes (including `GET /api/health`) are unaffected. PocketBase will auto-load this file from `pb_hooks/` on startup.
