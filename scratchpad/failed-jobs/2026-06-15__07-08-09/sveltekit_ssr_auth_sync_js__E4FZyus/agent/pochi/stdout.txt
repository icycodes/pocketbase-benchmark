Here is a summary of the changes and implementation:

1. **SvelteKit Server Hook (`src/hooks.server.js`)**:
   - Initializes a new PocketBase instance pointing to `http://127.0.0.1:8090` for every incoming request.
   - Loads the authentication state from the request cookie (`pb_auth`).
   - Verifies and refreshes the loaded auth state if it is valid. If the refresh fails, it clears the auth store.
   - Attaches the PocketBase instance to `event.locals.pb` so it is accessible to endpoints and page loads.
   - Resolves the event, and then serializes the updated auth store state back to the client using the response `set-cookie` header (with `secure: false` option to ensure compatibility with local HTTP/127.0.0.1 development).

2. **Login API Endpoint (`src/routes/api/login/+server.js`)**:
   - Accepts a `POST` request with JSON containing `email` and `password`.
   - Authenticates the user with PocketBase using `locals.pb.collection('users').authWithPassword(email, password)`.
   - Returns a `200 OK` JSON response. Since the SvelteKit server hook executes post-resolution, it automatically serializes the authenticated PocketBase state and appends the `set-cookie` header with the `pb_auth` cookie.

3. **Me API Endpoint (`src/routes/api/me/+server.js`)**:
   - Accepts a `GET` request.
   - Validates that the user is authenticated by checking `locals.pb.authStore.isValid` and the presence of the model.
   - Returns `401 Unauthorized` if the user is not authenticated.
   - Returns `200 OK` with `{"email": "<user_email>"}` if the user is authenticated.
