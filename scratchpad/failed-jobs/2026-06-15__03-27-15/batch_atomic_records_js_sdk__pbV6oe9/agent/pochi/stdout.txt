Created `/home/user/myproject/app.js` that atomically creates an `orders` record with N related `order_items` records using the PocketBase JS SDK batch API.

**Key design decisions:**

1. **Custom ID generation**: Since PocketBase's batch API doesn't support `@batch.{index}.id` cross-reference syntax, the script generates a custom 15-character alphanumeric ID (`[a-z0-9]{15}`) for the order record upfront, then references that ID in all `order_items` creation requests within the same batch.

2. **Batch API usage**: Uses `pb.createBatch()` + `batch.collection('orders').create()` + `batch.collection('order_items').create()` + `batch.send()` as required — no individual create requests.

3. **Fail mode**: When `--fail` is passed, an extra request targeting a non-existent collection (`/api/collections/nonexistent_collection/records`) is appended to the batch, causing a 404 error that triggers a full transactional rollback. The error is caught, `BATCH_ROLLED_BACK` is written to stderr, and the process exits with code 1.

**Verified behavior:**
- `node app.js --items 3` → exits 0, prints `ORDER:<15-char-id>` matching `^ORDER:[a-z0-9]{15}$`
- `node app.js --items 3 --fail` → exits 1, prints `BATCH_ROLLED_BACK` to stderr, no records persisted

- Created `order_items` correctly reference the order via the `order` relation field
