## Users

The pre-seeded `users` collection contains two accounts:

- `bob@example.com` (password: `BobPass1234`)

## How It Works

The primary filtering is enforced entirely by PocketBase's collection API rules. When a client subscribes to the `notifications` collection via SSE (`GET /api/realtime` → `POST /api/realtime` with `subscriptions: ["notifications"]`):

- **Authenticated user A** receives SSE `create` events only for records where `recipient = A's id`
- **Authenticated user B** receives SSE `create` events only for records where `recipient = B's id`
- **Anonymous users** (no auth header) receive zero `create` events, since `@request.auth.id` is empty and no record satisfies `recipient = ""`

This was verified by testing: creating 3 notifications with recipients `[A, B, A]` via the superuser REST API resulted in Alice receiving exactly 2 messages, Bob receiving exactly 1, and anonymous receiving 0.
