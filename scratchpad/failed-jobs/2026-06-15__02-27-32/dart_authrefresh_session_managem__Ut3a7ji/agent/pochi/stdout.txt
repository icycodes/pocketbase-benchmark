## How it works

### `login <email> <password>`
- Calls `pb.collection('users').authWithPassword(email, password)` via the official `pocketbase` pub package.
- Writes `session.json` in the current working directory:
  ```json
  { "token": "<jwt>", "record": { …RecordModel fields… } }
  ```
- Exits 0.

### `refresh`
- Reads and validates `session.json`; any parse error or missing/empty `token` key → prints `INVALID_SESSION` to stderr, exits 1.
- Restores the PocketBase auth store with the stored token + record so the SDK attaches the correct `Authorization` header.
- Calls `pb.collection('users').authRefresh()`.
- Persists the new token back to `session.json` only when it changed.
- Decodes the JWT payload (base64url, no signature verification needed) to extract the `exp` claim.
- Prints **exactly two lines** to stdout and nothing else:
  1. The authenticated user record id.
  2. The token `exp` as an integer epoch in seconds.
- Exits 0.
