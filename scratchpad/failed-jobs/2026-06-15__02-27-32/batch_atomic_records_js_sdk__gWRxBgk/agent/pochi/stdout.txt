## How it works

### Normal mode — `node app.js --items <N>`
1. Authenticates against PocketBase as superuser using `PB_URL`, `PB_ADMIN_EMAIL`, `PB_ADMIN_PASSWORD`.
2. Pre-generates a 15-character lowercase alphanumeric order ID client-side (matching PocketBase's own `[a-z0-9]{15}` pattern).
3. Opens a single `pb.createBatch()` and queues:
   - One `orders.create` with the pre-generated `id` and `customer = ZEALT_RUN_ID`
   - N `order_items.create` calls, each with `order = <orderId>`, a unique `product`, and `quantity = 1`
4. Sends the batch atomically via `batch.send()` — a single `POST /api/batch` request.
5. Prints `ORDER:<orderId>` to stdout and exits 0.

### Fail mode — `node app.js --items <N> --fail`
1. Same setup, but appends one extra `order_items.create` with `quantity: 0` — which violates the collection's `min: 1` / `onlyInt: true` constraint.
2. PocketBase rejects the entire batch transactionally; no records are written.
3. The catch block prints `BATCH_ROLLED_BACK` to stderr and exits 1.
