Created `/home/user/pocketbase/pb_hooks/sanitize_email.pb.js` with the following hook:

```js
onRecordCreateRequest((e) => {
    const email = e.record.get("email");
    if (email) {
        e.record.set("email", email.trim().toLowerCase());
    }
    e.next();
}, "users");

onRecordUpdateRequest((e) => {
    const email = e.record.get("email");
    if (email) {
        e.record.set("email", email.trim().toLowerCase());
    }
    e.next();
}, "users");
```

**How it works:**

- `onRecordCreateRequest` — fires before a new `users` record is validated and saved (POST `/api/collections/users/records`).
- `onRecordUpdateRequest` — fires before an existing `users` record is validated and saved (PATCH `/api/collections/users/records/:id`).
- Both handlers read the current `email` value, apply `.trim().toLowerCase()`, write it back with `e.record.set(...)`, then call `e.next()` to continue the request chain so PocketBase proceeds with validation and persistence.
- The second argument `"users"` scopes each hook exclusively to the `users` collection, so other collections are unaffected.

PocketBase automatically loads all `*.pb.js` files from `pb_hooks/` on startup — no additional registration is needed. Start the server with `./pocketbase serve --http=0.0.0.0:8090` and the sanitization will be active immediately.
